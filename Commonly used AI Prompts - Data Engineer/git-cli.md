# Git / CLI / Shell Prompts

## Git command help
```
I want to [git goal, e.g. "undo last commit but keep changes", "squash last 3 commits",
"move uncommitted work to a new branch", "resolve a rebase conflict"]. Give me the exact
git commands, what each does, and how to undo if it goes wrong.
```

## Explain git state
```
Explain what this git status/log/output means and what to do next:
[paste output]
```

## Fix a git mess
```
I'm stuck in git: [describe situation, e.g. "detached HEAD", "accidentally committed to
main", "pushed a bad commit", "merge conflict I don't understand"]. Walk me through fixing
it safely, step by step. Don't lose my work.
```

## Bash / shell one-liners
```
Write a [bash/zsh] one-liner (or short script) to [task, e.g. "find files larger than 100MB",
"bulk rename .txt to .md", "grep across files and count matches", "tail and filter logs"].
Explain each flag. Make it safe (no accidental overwrite).
```

## Regex
```
Write a regex to match [pattern, e.g. "emails", "dates in YYYY-MM-DD", "log lines with
ERROR and a timestamp"]. Target [grep / Python / sed]. Explain each part and give test
strings that should and shouldn't match.
```

## jq / JSON
```
Write a jq command to [extract/filter/transform] from this JSON: [paste sample]. Goal:
[describe output]. Explain the filter.
```

## cron
```
Give me the cron expression for "[schedule in plain English, e.g. 'every weekday at 6am',
'every 15 min during business hours']" and explain each field. Note timezone gotchas.
```

## Build a CLI script
```
Write a [bash/Python] script that [task]. Include argument parsing, help text, error
handling, and a dry-run option. Make it idempotent and safe to rerun.
```
