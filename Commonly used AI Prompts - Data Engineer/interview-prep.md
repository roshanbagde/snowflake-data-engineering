# Interview Prep Prompts

## SQL drills
```
Give me [5] SQL interview questions at [easy/medium/hard] on [topic]. Provide sample schema
+ data. Let me attempt first — put solutions and explanations in a separate section at the end.
```

## Mock technical interview
```
Act as an interviewer for a [Data Engineer] role. Ask me one [SQL / Python / data modeling /
pipeline design] question at a time, wait for my answer, then give feedback and a follow-up.
Increase difficulty as I go. Start now.
```

## System / pipeline design
```
Give me a data engineering system design question (e.g. "design a pipeline for X"). After I
answer, evaluate it on: requirements gathering, ingestion, storage, processing, scheduling,
data quality, scaling, and cost. Point out what a strong answer includes.
```

## Explain a concept for interviews
```
Explain [concept, e.g. "idempotency", "slowly changing dimensions", "partitioning vs
clustering", "CAP theorem", "incremental models"] the way I should explain it in an
interview: clear definition, why it matters, and a real example. Then quiz me on it.
```

## Behavioral (STAR)
```
Help me answer this behavioral question: "[paste question]". Ask me for the details, then
structure my answer using STAR (Situation, Task, Action, Result). Keep it under 2 minutes
spoken.
```

## Build STAR stories
```
Help me build 5 reusable STAR stories from my experience for a [DE] role, covering: conflict,
failure, leadership, ambiguity, and impact. Ask me about my projects, then draft each story.
```

## Resume / project bullets
```
Rewrite these resume bullets to be impact-focused (action verb + what + measurable result).
Tailor to a [Data Engineer] role. Original: [paste].
```

## Company / role research
```
I'm interviewing for [role] at [company type]. Suggest likely interview topics, smart
questions to ask them, and how to position my background [describe] for this role.
```
