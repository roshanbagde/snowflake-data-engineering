❄️ Snowflake Dynamic Tables — Cheat Sheet

verified vs docs · May 2026

⚙️ Anatomy

CREATE OR REPLACE DYNAMIC TABLE my_table
  TARGET_LAG   = '5 minutes' | DOWNSTREAM   -- min 60s
  WAREHOUSE    = transform_wh
  REFRESH_MODE = AUTO | FULL | INCREMENTAL  -- AUTO = default
  INITIALIZE   = ON_CREATE | ON_SCHEDULE
  IMMUTABLE WHERE (order_ts < CURRENT_TIMESTAMP() - INTERVAL '30 days')
  BACKFILL FROM history_table
AS SELECT ... FROM base;

⏱️ TARGET_LAG = DOWNSTREAM

  • Freshness goal, not a cron.
  • Set a real number only on leaf / mart tables.
  • DOWNSTREAM on intermediates → they self-tune to consumers.
  • Stops paying to refresh tables nobody needs fresh.

🔁 Refresh modes — the trap

  • AUTO picks INCREMENTAL/FULL once at create, then locks.
  • Unsupported SQL → silent FULL refresh forever.
  • Audit refresh_mode_reason after every deploy.
  • Force INCREMENTAL = loud error instead of silent FULL.

🔒 IMMUTABLE WHERE recent

  • Freeze rows that match → skipped on every refresh.
  • 1st refresh ignores it; later refreshes touch only mutable rows.
  • Works in incremental and full mode.
  • No subqueries / UDFs / non-deterministic fns.
  • Check via METADATA$IS_IMMUTABLE.

📦 BACKFILL FROM recent

  • Zero-copy adopt computed rows (no recompute).
  • Requires an IMMUTABLE WHERE (frozen rows only).
  • Clustering keys must match the source.
  • Migrate pipelines / evolve a definition cheaply.

📊 Monitor

  • SHOW DYNAMIC TABLES — quick status
  • DYNAMIC_TABLES() — lag ratios (7d)
  • ..._REFRESH_HISTORY() — per refresh (7d)
  • ..._GRAPH_HISTORY() — the DAG
  • ACCOUNT_USAGE view — 365d trends

🛠️ Manage

ALTER DYNAMIC TABLE t
  SET TARGET_LAG='1 hour';
ALTER ... SUSPEND | RESUME;
ALTER ... REFRESH;   -- now
CREATE ... CLONE t;
SELECT GET_DDL('TABLE','t');

💰 Cost levers (highest impact first)

  • Loosen TARGET_LAG; DOWNSTREAM on intermediates
  • Confirm INCREMENTAL (don't pay for silent FULL)
  • Write incremental-friendly SQL
  • IMMUTABLE WHERE on append-only / time-partitioned data
  • BACKFILL FROM instead of recomputing history
  • Right-size the warehouse (watch for spill)

🆚 vs alternatives

  • Dynamic table: declarative multi-step + SLA
  • Mat. view: single-table simple aggregate
  • Stream+Task: imperative / custom MERGE
  • View: always-fresh, compute on read
