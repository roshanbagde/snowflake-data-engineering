# ThinkSpark

A critical thinking and character development trainer for kids aged 5–18. Manage a question bank, generate new questions with AI, build classroom sessions, and run a projector-ready classroom view.

AI generation is optional — the app works fully offline with the built-in question bank.

---

## Features

- **Question Bank** — 115+ seed questions across 10 topics and 5 age groups; add, edit, and favourite your own
- **⭐ Favourites** — star questions and filter to favourites by default
- **JSON Import** — import questions generated by any external AI model; upload a file or paste JSON directly
- **Duplicate Detection** — automatically skips questions already in your bank on import; scan and clean existing duplicates
- **AI Generator** — generate batches of questions via any LLM; review and save selectively
- **Session Builder** — create dated sessions, pick questions, export as Markdown
- **Classroom View** — projector-ready full-screen display with hint and discussion reveal
- **8-Week Curriculum** guide built in
- Works with **Anthropic Claude, OpenAI, Google Gemini, Groq, Mistral, NVIDIA NIM, and Ollama**

## Setup

```bash
cd think-spark

python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Optional: add API key(s) for AI generation
cp .env.example .env

./start.sh                      # opens http://localhost:8505
```

To stop: `./stop.sh`

The app works without any API key using **Ollama** (local models) or by browsing the built-in question bank directly.

## Importing Questions from Any AI Model

You can generate questions outside the app (using ChatGPT, Claude, Gemini, or any model) and import them in bulk.

**Step 1 — Get the prompt**

Open the **Question Bank** tab → **Upload Questions from JSON** → expand **"How to generate questions with any AI model"**. Copy the ready-made prompt and paste it into any AI chat.

**Step 2 — Import the output**

Paste the JSON the model returns directly into the text area, or save it as a `.json` file and upload it. The app will:
- Validate every question (topic, age group, level, type)
- Flag duplicates already in your bank and skip them automatically
- Show a preview of new questions before saving

**Required JSON format**

```json
[
  {
    "topic": "⚖️ Ethics & Values",
    "age_group": "Builders",
    "level": "medium",
    "type": "dilemma",
    "question_text": "You find a phone on the street with no lock screen. What do you do?",
    "context": "You are walking home from school.",
    "hint": "Think about how the owner might be feeling right now.",
    "sample_answer": "Hand it in to the nearest police station or try to contact someone in the phone's contacts.",
    "discussion_points": [
      "Why is it important to return lost property?",
      "What would you want someone to do if they found your phone?"
    ],
    "follow_up_questions": [
      "Would your answer change if the phone was very old and broken?"
    ],
    "tags": "honesty,integrity,ethics"
  }
]
```

**Valid values**

| Field | Options |
|---|---|
| `topic` | `🧠 Critical Thinking` · `❤️ Emotional Intelligence` · `💬 Communication` · `🤝 Social Skills & Teamwork` · `🏆 Leadership & Decisions` · `🎨 Creativity & Problem Solving` · `⚖️ Ethics & Values` · `🪞 Self-Awareness` · `💰 Financial Literacy` · `🌍 Environment & Society` |
| `age_group` | `Seedlings` · `Explorers` · `Builders` · `Challengers` · `Leaders` |
| `level` | `simple` · `medium` · `hard` |
| `type` | `scenario` · `dilemma` · `reflection` · `debate_starter` · `creative` · `what_if` · `puzzle` |

`context`, `hint`, `sample_answer`, `discussion_points`, `follow_up_questions`, and `tags` are optional.

## Finding and Removing Duplicates

Open **Question Bank** → **Find & Remove Duplicates** → click **Scan for duplicates**.

- If duplicates exist, use **Delete all extras** to keep the oldest copy of each and remove the rest in one click.
- Or expand each group to review and delete individual copies.

## Configuration

AI question generation requires at least one LLM API key. Copy `.env.example` → `.env`:

| Variable | Provider |
|---|---|
| `ANTHROPIC_API_KEY` | Claude |
| `OPENAI_API_KEY` | GPT-4o |
| `GOOGLE_API_KEY` | Gemini |
| `GROQ_API_KEY` | Groq (free tier available) |
| `MISTRAL_API_KEY` | Mistral |
| `NVIDIA_API_KEY` | NVIDIA NIM |
