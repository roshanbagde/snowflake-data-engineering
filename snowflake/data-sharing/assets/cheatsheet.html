🔑 Provider — direct share

CREATE SHARE sales_share;
GRANT USAGE  ON DATABASE sales_db
  TO SHARE sales_share;
GRANT USAGE  ON SCHEMA sales_db.public
  TO SHARE sales_share;
GRANT SELECT ON VIEW …orders_v
  TO SHARE sales_share;
ALTER SHARE sales_share
  ADD ACCOUNTS = org1.consumer1;

📥 Consumer

CREATE DATABASE shared_sales
  FROM SHARE org1.prov.sales_share;
GRANT IMPORTED PRIVILEGES
  ON DATABASE shared_sales
  TO ROLE analyst;
SELECT * FROM
  shared_sales.public.orders_v;
  • Read-only, 1 DB / share
  • No onward re-sharing

🎯 Per-consumer rows

CREATE SECURE VIEW v AS
SELECT o.* FROM orders o
JOIN entitlements e
  ON o.region = e.region
 AND e.acct =
   CURRENT_ACCOUNT();
  • CURRENT_ACCOUNT() = the consumer
  • Always share SECURE views

🌍 Auto-fulfillment

  • Cross-region/cloud via a listing
  • Replicates to a managed SSA per region, on demand
  • Not on trial accounts
  • No secure views over other DBs
  • Cost = storage + compute + transfer

🔁 Replication vs Failover

Targetread-onlypromotable

  • *DB + share repl = all editions
  • Inbound shares don't replicate

🛟 DR for sharing

CREATE FAILOVER GROUP fg
 OBJECT_TYPES=DATABASES,
   SHARES,LISTINGS
 ALLOWED_DATABASES=db
 ALLOWED_SHARES=sh
 ALLOWED_ACCOUNTS=org.dr;
  • DB DR ≠ share/listing DR
  • All refs in one group (all-or-nothing)
  • Failover → consumers keep access

⚠️ Top limitations

• Direct share = same region + cloud only
• Shared objects read-only; 1 DB per share
• No transitive re-sharing		 • Auto-fulfillment not on trials; no cross-DB secure views
• Cloud-marketplace accounts fulfill within that cloud only
• Refresh frequency drives auto-fulfill cost		 • Failover/promotion needs Business Critical+
• Inbound shares can't be replicated
• BCDR excludes draft / stage-backed / paid / Native App listings
